# IMS Electron

Inventory Management System built with React + Electron + SQLite.

## Goal of this guide

This file explains how someone can clone this repository and fully recreate a working project environment, including development mode and production build artifacts.

## What is stored in this repo

- Source code in `src/`
- Electron main process code in `src/main/`
- React public assets in `public/`
- Dependency lock file in `package-lock.json`

## What is not stored in this repo

- `node_modules/` (reinstall after clone)
- `build/` (generated by React build)
- `dist/` (generated installer/AppImage output)

These generated folders are intentionally ignored and can be recreated from source.

## Prerequisites

- Git
- Node.js (recommended LTS, Node 18+)
- npm (comes with Node.js)
- Linux/Windows/macOS environment that can run Electron

## 1. Clone the repository

```bash
git clone https://github.com/Calvinkev/IMS.git
cd IMS/ims-electron
```

## 2. Install dependencies

```bash
npm install
```

This restores `node_modules/` from `package-lock.json`.

## 3. Run the project in development mode

```bash
npm run electron-dev
```

What this does:
- Starts React dev server on `http://localhost:3000`
- Launches Electron after the dev server is ready

## 4. Build production frontend files

```bash
npm run build
```

This recreates the `build/` folder.

## 5. Run Electron using local production build

```bash
IMS_FORCE_LOCAL_BUILD=1 npm run electron
```

This launches Electron and forces it to use `build/index.html` locally.

## 6. Recreate release artifacts

Linux AppImage:

```bash
npm run dist:linux
```

Windows installer:

```bash
npm run dist:win
```

General distribution build:

```bash
npm run dist
```

These commands recreate the `dist/` output.

## 7. Database behavior after clone

The app uses SQLite and auto-creates its database on first run.

- DB file name: `inventory.db`
- Stored in Electron `userData` path (not inside this repo)
- Default bootstrap login created automatically if no admin user exists:
  - Username: `admin`
  - Password: `admin123`

Because the DB is outside the repo, each cloned setup starts with a fresh local database unless a backup is imported.

## Optional: tracking built AppImage with Git LFS

This repo is configured to track `dist/*.AppImage` with Git LFS.

If you need to push AppImage files:

```bash
git add -f dist/*.AppImage
git commit -m "Add AppImage artifact"
git push
```

If push fails with an LFS budget error, repository/account LFS storage or bandwidth must be increased in GitHub billing before the artifact can upload.

## Quick recovery checklist (for new clones)

```bash
git clone https://github.com/Calvinkev/IMS.git
cd IMS/ims-electron
npm install
npm run electron-dev
```

For production package output after that:

```bash
npm run dist:linux
```
